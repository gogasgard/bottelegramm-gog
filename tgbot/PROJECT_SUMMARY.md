## 🔧 Технические детали

### Зависимости (requirements.txt)
- `aiogram==3.13.1` - Telegram Bot фреймворк
- `yt-dlp` - Загрузка видео
- `sqlalchemy==2.0.36` - ORM
- `aiosqlite` - Асинхронная SQLite
- `ffmpeg-python` - Обработка видео
- `aiofiles` - Асинхронная работа с файлами
- `python-dotenv` - Загрузка .env
- `loguru` - Логирование

### Основные классы и функции

**Config** (`app/config.py`)
- Загрузка конфигурации из .env
- Валидация обязательных параметров
- Создание необходимых директорий

**Database** (`app/database/database.py`)
- Методы для работы с пользователями
- Методы для работы с загрузками
- Асинхронные операции с БД

**VideoDownloader** (`app/services/downloader.py`)
- `get_video_info()` - получение информации о видео
- `get_available_qualities()` - список доступных качеств
- `download_video()` - загрузка видео
- `download_audio()` - загрузка аудио

**VideoProcessor** (`app/services/video_processor.py`)
- `compress_video()` - сжатие через ffmpeg
- `split_video()` - разбивка на части
- `check_if_needs_processing()` - проверка размера

**QueueManager** (`app/services/queue_manager.py`)
- Управление очередями для каждого пользователя
- Последовательная обработка задач

**ProgressTracker** (`app/services/progress.py`)
- Отслеживание прогресса загрузки
- Обновление сообщений в Telegram
- Форматирование прогресс-бара

## 🔄 Процесс работы бота

### 1. Регистрация пользователя
```
Пользователь → /start → Ввод пароля → Регистрация в БД → Доступ разрешен
```

### 2. Загрузка видео
```
URL → Валидация → Получение info → Выбор качества → Выбор субтитров
→ Загрузка → Проверка размера → Обработка (если нужно)
→ Отправка пользователю → Пересылка в канал → Удаление с сервера
→ Сохранение в БД → Обновление статистики
```

### 3. Обработка больших файлов
```
Файл >2 ГБ → Автоматическое сжатие → Проверка размера
→ Если все еще >2 ГБ: Разбивка на части → Отправка частей
```

## ⚙️ Особенности реализации

### Асинхронность
Все операции ввода-вывода асинхронные:
- База данных (async SQLAlchemy)
- Telegram API (aiogram)
- Файловые операции (aiofiles)
- yt-dlp обернут через `asyncio.to_thread()`

### Middleware авторизации
Проверяет каждое сообщение и callback на авторизацию.
Пропускает только `/start` для незарегистрированных.

### Кеширование информации о видео
Информация о видео хранится в памяти (router._video_info_cache)
по message_id для быстрого доступа при нажатии кнопок.

### Прогресс-бар
Обновляется каждые 3 секунды для экономии API запросов.
Использует hooks yt-dlp для отслеживания прогресса.

### Очистка файлов
Все временные файлы удаляются сразу после отправки
для экономии дискового пространства.
