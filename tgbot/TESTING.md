# 🧪 Инструкции по тестированию

## Подготовка к тестированию

### 1. Создание бота в Telegram

1. Найдите [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Введите имя бота (например: "My Video Downloader")
4. Введите username бота (должен заканчиваться на `bot`, например: `my_video_dl_bot`)
5. Скопируйте полученный токен

### 2. Создание тестового канала

1. Создайте новый канал в Telegram
2. Добавьте вашего бота в канал как администратора
3. Дайте боту права на "Публикация сообщений"
4. Для получения ID канала:
   - Перешлите любое сообщение из канала боту [@userinfobot](https://t.me/userinfobot)
   - Скопируйте ID канала (например: `-1001234567890`)

### 3. Настройка .env файла

```env
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz  # Ваш токен от BotFather
ADMIN_PASSWORD=test_password_123                  # Тестовый пароль
CHANNEL_ID=-1001234567890                         # ID вашего канала
DATABASE_PATH=data/database.db
DOWNLOADS_PATH=data/downloads
LOG_LEVEL=DEBUG  # Для детального логирования при тестировании
```

## Запуск для тестирования

### Локальный запуск (с виртуальным окружением)

```bash
# Активация виртуального окружения
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# Запуск бота
python -m app.main
```

### Запуск через Docker

```bash
# Сборка и запуск
docker-compose up --build

# Просмотр логов в реальном времени
docker-compose logs -f

# Остановка
docker-compose down
```

## Известные проблемы и решения

### Проблема: "Error getting video info"

**Причины:**
- Видео недоступно (удалено, приватное)
- Ограничения по региону
- Устаревшая версия yt-dlp

**Решение:**
```bash
pip install -U yt-dlp
```

### Проблема: "Не удалось переслать в канал"

**Причины:**
- Бот не добавлен в канал
- Недостаточно прав
- Неправильный CHANNEL_ID

**Решение:**
1. Проверьте, что бот в канале
2. Дайте права администратора
3. Убедитесь, что ID канала правильный (с минусом)

### Проблема: Бот не отвечает

**Причины:**
- Неправильный токен
- Проблемы с сетью
- Бот уже запущен в другом месте

**Решение:**
1. Проверьте BOT_TOKEN в .env
2. Проверьте интернет-соединение
3. Остановите другие экземпляры бота
