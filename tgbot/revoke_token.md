# 🔴 Радикальное решение конфликта

Если конфликт продолжается, выполните следующие шаги:

## Вариант 1: Подождать 5-10 минут

Telegram иногда держит соединение до 5 минут после закрытия. Просто подождите и попробуйте снова.

## Вариант 2: Отозвать и получить новый токен

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/mybots`
3. Выберите вашего бота
4. Нажмите `API Token`
5. Нажмите `Revoke current token`
6. Скопируйте новый токен
7. Обновите `BOT_TOKEN` в файле `.env`
8. Запустите бота снова

## Вариант 3: Проверьте другие устройства

**ВАЖНО:** Убедитесь, что бот не запущен:
- На другом компьютере
- На VPS/сервере
- В облачном хостинге (Heroku, AWS, и т.д.)
- В другом терминале на этом же компьютере

## Вариант 4: Принудительное удаление webhook

Выполните вручную:

```bash
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/deleteWebhook?drop_pending_updates=true"
```

Замените `<YOUR_BOT_TOKEN>` на ваш токен.

## Вариант 5: Используйте Task Manager

1. Откройте Task Manager (Ctrl+Shift+Esc)
2. Найдите все процессы `python.exe`
3. Завершите их принудительно (End Task)
4. Подождите 2 минуты
5. Запустите бота снова

---

## Проверка статуса бота

Используйте этот URL в браузере (замените TOKEN):

```
https://api.telegram.org/bot<YOUR_TOKEN>/getWebhookInfo
```

Должно быть: `"url": ""`

---

**После выполнения любого из этих шагов подождите минимум 2 минуты перед запуском бота.**
