# 🎬 Telegram Video Downloader Bot

Телеграм-бот для загрузки видео с популярных платформ: YouTube, VK Видео, Rutube и TikTok.

## ✨ Возможности

- 📹 **Загрузка видео** с выбором качества
- 🎵 **Извлечение аудио** в формат MP3
- 📝 **Загрузка субтитров**
- 🗜️ **Автоматическое сжатие** видео >2 ГБ
- ✂️ **Разбивка на части** больших файлов
- 📊 **Статистика** использования
- 🔐 **Авторизация по паролю**
- 📢 **Автоматическая пересылка** видео в канал
- 🎯 **Прогресс загрузки** в реальном времени

## 🌐 Поддерживаемые платформы

- ▶️ YouTube (youtube.com, youtu.be)
- 🎵 VK Видео (vk.com, vk.ru)
- 📺 Rutube (rutube.ru)
- 🎬 TikTok (tiktok.com)

## 🛠️ Технологии

- **Python 3.11+**
- **aiogram 3.x** - асинхронный фреймворк для Telegram Bot API
- **yt-dlp** - универсальная библиотека для загрузки видео
- **SQLAlchemy + SQLite** - база данных для пользователей и статистики
- **ffmpeg** - обработка и конвертация видео
- **Docker** - контейнеризация приложения

## 📋 Требования

- Python 3.11 или выше
- ffmpeg
- Docker и Docker Compose (опционально)

## 🚀 Установка

### Вариант 1: С использованием Docker (рекомендуется)

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd tgbot
```

2. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

3. Отредактируйте `.env` и заполните необходимые данные:
```env
BOT_TOKEN=ваш_токен_бота_от_BotFather
ADMIN_PASSWORD=ваш_пароль_для_доступа
CHANNEL_ID=-1001234567890  # ID канала для пересылки видео
```

4. Запустите бота:
```bash
docker-compose up -d
```

5. Просмотр логов:
```bash
docker-compose logs -f
```

### Вариант 2: Без Docker

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd tgbot
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Установите ffmpeg:
   - **Ubuntu/Debian**: `sudo apt-get install ffmpeg`
   - **macOS**: `brew install ffmpeg`
   - **Windows**: Скачайте с [официального сайта](https://ffmpeg.org/download.html)

5. Создайте файл `.env`:
```bash
cp .env.example .env
```

6. Отредактируйте `.env` и заполните данные

7. Запустите бота:
```bash
python -m app.main
```

## ⚙️ Конфигурация

### Переменные окружения (.env)

| Переменная | Описание | Обязательна |
|------------|----------|-------------|
| `BOT_TOKEN` | Токен бота от @BotFather | ✅ |
| `ADMIN_PASSWORD` | Пароль для регистрации в боте | ✅ |
| `CHANNEL_ID` | ID канала для пересылки видео | ✅ |
| `DATABASE_PATH` | Путь к файлу БД | ❌ (по умолчанию: data/database.db) |
| `DOWNLOADS_PATH` | Путь для временных файлов | ❌ (по умолчанию: data/downloads) |
| `LOG_LEVEL` | Уровень логирования | ❌ (по умолчанию: INFO) |

### Как получить ID канала?

1. Создайте канал в Telegram
2. Добавьте бота в канал как администратора с правами на публикацию
3. Перешлите любое сообщение из канала боту [@userinfobot](https://t.me/userinfobot)
4. Скопируйте ID канала (например: -1001234567890)

## 📖 Использование

### Регистрация

1. Найдите вашего бота в Telegram
2. Отправьте команду `/start`
3. Введите пароль (из `.env` файла)

### Загрузка видео

1. Отправьте боту ссылку на видео
2. Выберите желаемое качество из предложенных вариантов
3. Выберите, нужны ли субтитры
4. Дождитесь загрузки и отправки файла

### Загрузка аудио

1. Отправьте ссылку на видео
2. Нажмите кнопку "🎵 Только аудио (MP3)"
3. Дождитесь загрузки

### Команды бота

- `/start` - Начать работу / Регистрация
- `/help` - Справка по использованию
- `/stats` - Ваша статистика (количество загрузок, объем)
- `/cancel` - Отменить текущую загрузку
- `/queue` - Показать очередь загрузок

## 🔧 Обработка больших файлов

Telegram имеет ограничение на размер файлов в **2 ГБ**. Бот автоматически обрабатывает большие файлы:

1. **Автоматическое сжатие**: Бот попытается сжать видео до размера <2 ГБ
2. **Разбивка на части**: Если после сжатия файл все еще большой, он будет разбит на части

## 📊 База данных

Бот использует SQLite для хранения:

- **Пользователи**: user_id, username, дата регистрации, статистика
- **Загрузки**: история загрузок, информация о видео, время загрузки

## 📝 Логирование

Логи сохраняются в двух местах:

1. **Консоль**: Вывод в реальном времени (цветной)
2. **Файл**: `logs/bot.log` с ротацией:
   - Новый файл создается при достижении 10 МБ
   - Старые логи хранятся 7 дней
   - Автоматическое сжатие в ZIP

Уровни логирования: DEBUG, INFO, WARNING, ERROR, CRITICAL

## 🐛 Решение проблем

### Бот не запускается

1. Проверьте правильность `BOT_TOKEN` в `.env`
2. Убедитесь, что установлены все зависимости
3. Проверьте логи: `docker-compose logs` или `logs/bot.log`

### Ошибка "Не удалось переслать в канал"

1. Убедитесь, что бот добавлен в канал
2. Проверьте, что бот имеет права администратора
3. Убедитесь, что `CHANNEL_ID` указан правильно (включая минус для каналов)

### Ошибка загрузки видео

1. Обновите yt-dlp: `pip install -U yt-dlp`
2. Проверьте, что видео доступно (не удалено, не приватное)
3. Проверьте логи для подробностей

### Недостаточно места на диске

1. Временные файлы автоматически удаляются после отправки
2. Очистите директорию `data/downloads/` вручную
3. Настройте автоматическую очистку старых файлов
